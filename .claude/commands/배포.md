# `/배포` — dev→main 검증 + PR + npm publish

npm 라이브러리 배포 절차. AGENTS.md §배포 워크플로우 정합. STEP 순차 실행.

## 주의

- `git push --force` / `git reset --hard` 금지.
- uncommitted 변경 있으면 중단(먼저 `/push`).
- 테스트 실패 시 배포 금지.
- `.npmrc`·시크릿 출력 노출 금지.
- 배포는 `v*` 태그 push 로 트리거(OIDC 자동 publish) — 배포 승인 = 태그 push 시점(PM 결정).

## STEP 0 — 사전 확인

```bash
git status --short          # 비어있어야 함. 아니면 중단 → /push 먼저
git branch --show-current   # dev 여야 정상
```
- 미커밋 있으면: `[STEP 0] ❌ 미커밋 변경 — /push 먼저` 후 중단.

## STEP 1 — E2E Smoke PreFlight (배포 차단 게이트)

이번 배포 변경이 아래 trigger 영역에 영향을 주면, 대표 경로(critical path) 통합 테스트를 **신설/갱신**했는지 점검.

**Trigger 영역**:
1. OOXML 생성(`src/helpers/xml-builder.js`, `src/docx-document.js`, `src/schemas/`) — 표·테두리·리스트·이미지·문단/런 속성
2. 변환 진입(`src/helpers/render-document-file.js`, `src/helpers/html-parser.js`, `src/vdom/`) — HTML→가상 DOM→OOXML
3. 공개 API(`index.js`, `src/html-to-docx.js`) — `generateContainer` 시그니처·옵션·헤더/푸터

```bash
git diff --name-only main..dev -- src/ index.js | head -30
```
- 변경 있음 + `tests/`에 대표 경로 테스트 신설/갱신 → `[STEP 1] ✅ trigger 변경 + 테스트 갱신 확인`
- 변경 있음 + 미갱신 → 🔴 `[STEP 1] 🔴 trigger 변경 but 테스트 미갱신 — 대표 경로 테스트 신설 후 재진입 (계속? y/N)` PM `y` 아니면 중단.

## STEP 2 — 전체 검증

```bash
npm run build && npm run test:unit    # 빌드 3타깃 + 625+ 전부 통과해야
```
- 실패 시 `[STEP 2] ❌ 테스트 실패 — 배포 중단`.
- 가능 시 대표 HTML 변환 산출물 Word 열림 수동 확인(`etc/test source/` → `.docx` 생성 후 열기).

## STEP 3 — 버전 bump(dev) → dev → main PR

> 🔒 `main` 은 GitHub branch protection 으로 **직접 push 차단**(PR 필수·enforce_admins). 버전 bump 커밋은 반드시 PR 에 실어 보낸다 — main 직접 push 금지.

```bash
npm version <patch|minor|major> --no-git-tag-version   # package.json + lock 만 갱신(커밋·태그 X)
git commit -am "chore(release): v<X.Y.Z>"
git push origin dev          # dev 최신화(버전 bump 포함)
gh pr create --base main --head dev \
  --title "release: v<X.Y.Z> — <요약>" \
  --body "변경 요약 / 검증 결과(테스트 N개) / 대표 경로 / Word 열림 확인"
```
- PR 에 버전 bump 커밋 포함됨. PR URL 출력 → PM 검토·머지 대기(또는 PM 명시 시 `gh pr merge --squash`).

## STEP 4 — 태그 push → OIDC 자동 publish (main 머지 후)

`v*` 태그 push 가 곧 배포 트리거 — `.github/workflows/publish.yml`(OIDC Trusted Publisher)이 토큰 없이 `npm publish --provenance` 수행. 로컬 수동 publish 불필요.

```bash
git checkout main && git pull origin main
git tag v<X.Y.Z> && git push origin v<X.Y.Z>          # → publish.yml(OIDC) trigger. main 직접 push X
gh run list --workflow=publish.yml --limit=1          # 실행 ID 확인
gh run watch <id>                                     # Actions 진행/결과 확인
npm view @shyang1012/docx-convert version             # 게시 확인
```
- 🔴 `git push origin main` 직접 push 금지(branch protection 차단) — 버전 bump 는 STEP 3 PR 로 이미 main 에 반영됨.
- **publish 는 Actions(OIDC)가 자동 수행** — 로컬 토큰·`npm publish` 불필요. 배포 승인 = 태그 push 시점(PM 결정).
- (Actions 실패·긴급 시 fallback) 로컬 수동: `npm run build && npm publish --access public`(Automation 토큰 필요).

## STEP 5 — 마무리

```bash
git checkout dev && git merge main   # main 의 version bump 를 dev 에 backsync
git push origin dev
```

완료 출력:
```
[배포 완료]
- PR: {url} (머지 {해시})
- npm: @shyang1012/docx-convert@{버전} 게시
- 검증: 테스트 {N}개 + 대표 경로 {범위}
```
