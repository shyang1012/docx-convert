# `/시작` — 세션 컨텍스트 로드 + 상태 보고

프로젝트 컨텍스트를 로드하고 현재 작업 상태를 보고하라.

## STEP 1 — 운영 정책 상기

`AGENTS.md` 를 읽어 핵심 정책을 상기한다: dev 개발 / main 배포(검증+PR) / beads 이슈 / local-first / 보안(.npmrc·시크릿). 아키텍처 상세는 `CLAUDE.md`.

## STEP 2 — 브랜치 + 최근 이력

```bash
git branch --show-current        # dev 정상
git log --oneline -5
git status --short                # 미커밋 잔여 확인
```

## STEP 3 — beads 컨텍스트 로드 + 작업 현황

`bd prime` 안내에 따라 **가볍게** 로드한다(메모리 본문 일괄 주입 X — 인덱스 + 필요분만):

```bash
bd prime            # 로드 순서·핵심 규칙(가벼운 골격)
bd memories         # 메모리 인덱스(키+요약) — 본문은 bd recall <key>
bd ready            # 착수 가능 이슈
```
- 인덱스의 `session-*` 중 **최근 3개만** `bd recall <key>` 로 펼쳐 세션 맥락 확인. 나머지 세션은 펼치지 않음.
- 특정 주제 필요 시에만 `bd recall <key>`(예: `memory-policy`/`fork-origin`/`toolchain`).
- 실행 실패 메시지가 확인된 경우에만 건너뜀. 🔴 미설치·미초기화 가정 금지(실제 실행 결과로만 판단).

## STEP 4 — 상태 보고

```
[세션 시작] 컨텍스트 로드 완료
- 브랜치: {현재} / 미커밋: {N건 또는 없음}
- 최근 작업: {git log 최근 1~2개}
- 착수 가능(bd ready): {Top 3 또는 0건}
- 메모: {미배포 변경·진행 중 작업이 있으면 한 줄}
```
